Git（一）：第一次完整的基本操作

转载：https://blog.csdn.net/qq_35206244/article/details/97698815

一、基本概念

1.工作流程

  1. 克隆 Git 资源作为工作目录
  2. 在克隆下来的资源上添加或修改
  3. 提交修改前，先更新资源，防止要提交的修改和其他人的修改冲突
  4. 提交修改

2.概念

  • 工作区（Workspace）：本地能看到的目录。
  • 暂存区（Index / Stage）：一般存放在 .git 目录下的 index 文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。
  • 版本库（Repository）：隐藏目录 .git，这个不算在工作区中，而是 Git 的版本库。亦称为本地仓库。
  • 远程仓库（Remote）

3.SVN与Git的主要区别

  • SVN是集中式版本控制系统
    • 版本库是集中放在中央服务器的。
    • 工作流程：从中央服务器拉取最新的版本，干活后，提交自己的内容到中央服务器
    • 缺点：需要联网，对网络依赖大
  • Git是分布式版本控制系统
    • 没有中央服务器
    • 每个人的电脑就是一个完整的版本库
    • 工作的时候就不需要联网了，因为版本都是在自己的电脑上
    • 协作：互相推送，可以看到对方的修改

基本流程：add->commit->push

1.建立本地仓库

如果是需要拉取一个项目则不需要建立，因为git clone下来的文件夹包含了git信息【.git文件夹】，即为本地仓库，当作本地项目直接使用即可。

2.推送文件至远程仓库
