第1章 Java程序设计概述

​ 本章简要的介绍Java语言的发展历史

1.1 Java程序设计平台

介绍

  • Java有许多优秀的特性。
  • 比之前的多种语言引起轰动的原因：Java是一个完整的平台，有一个庞大的库。比如：很多可重用的代码和一个提供诸如安全性、跨操作系统的可移植性以及自动垃圾回收等服务的执行环境。
  • 具有令人赏心悦目的语法和易于理解的语义。

1.2 Java“白皮书”的关键术语

Java 的设计者已经编写了颇有影响力的“ 白皮书”，用来解释设计的初衷以及完成的情况，并且发布了一个简短的摘要，11个关键术语来组织。

11个关键术语

  1. 简单性：

    • 对比C++而已简单。类似C++但是又剔除了C++的许多难以使用、易混淆的特性。
    • 程序小。能够在小型机器上独立运行。

  2. 面向对象：

    • 重点放在数据（即对象）和对象的接口上
    • 比如：工匠始终关注所要制作的椅子，然后才是工具

  3. 分布式

    • Java 有一个丰富的例程库，用于处理像 HTTP 和 FIT 之类的 TCP/IP 协议。Java 应用程序能够通过 URL 打开和访问网络上的对象，其便捷程度就好像访问本地文件一样。
    • 总：分布式体现在“好像访问本地文件一样”，本地文件分布在其他网络主机上。

  4. 健壮性

    • 进行早期的问题检测、 后期动态的 （运行时）检测，并消除了容易出错的情况
    • Java 和 C++ 最大的不同在于 Java 采用的指针模型可以消除重写内存 和损坏数据的可能性
    • 总：问题检测更方便，定位bug更快，使得最终的程序更加可靠。

  5. 安全性

    • 不可信代码在一个沙箱环境中执行，在这里它不会影响主系统。用户可以确信不会发生不好的事情， 因为 Java 代码不论来自哪里，都不能脱离沙箱。
    • 遗憾没有想象中那么成功，但在那个时代以及是超前。

  6. 体系结构中立

    • 编译器生成一个体系结构中立的目标文件格式，只要有Java运行时系统，这些编译后的代码可以在多处处理机上运行。
    • 编译器通过使生成的字节码指令不依赖于计算机体系结构来实现
    • 精心设计的字节码不仅 可以很容易地在任何机器上解释执行，而且还可以动态地翻译成本地机器代码。
    • 即为虚拟机生成代码，这种技术在Java之前以及运用。
    • 总：一次编译多处编译运行

  7. 可移植性

    • 与 C 和 C++ 不同，Java 规范中没有“ 依赖具体实现” 的地方，基本教据类型的大小以及有关运算都做了明确的说明。
      • C/C++中，int可能是16位也可能是32位
      • Java中数据类型大小固定，二进制数据以固定的格式进行存储和传输， 消除了字节顺序的困扰，更易于移植，
    • Java程序中会作为系统组成部分的类库，定义了可移植的接口，除了用户界面有关的部分，其他的Java库都能很好地支持平台独立性。
      • 你可以处理文件、 正则表达式、XML、日期和时间、数据库、网络连接、线程等，而不用操心 底层操作系统。不仅程序是可移植的，Java API 往往也比原生 API 质量更高。
    • 总：数据类型固定易于移植、Java接口在不同平台有其不同的实现，使得我们无需关心底层操作系统，无需大改代码。

  8. 解释性

    • Java 解释器可以在任何移植了解释器的机器上执行 Java 字节码。由于链接是一 个增量式且轻量级的过程， 所以， 开发过程也变得更加快捷，更加具有探索性。
    • 总：可以在其他机器上解释执行

  9. 高性能

    • 字节码可以（在运行时刻）动态地翻译成对应运行这个应用的特定 CPU 的机器码–即时编译
    • 总：即时编译，可以监控执行过程，便于优化

  10. 多线程

    • 多线程可以带来更好的交互响应和实时行为
      • 因为单个处理器速度有限，我们不再追求更快的处理器，而是着眼于获得更多的处理器，且要让它们一直保持工作
      • Java在当时就已经超前，是第一个支持并发程序设计的主流语言

  11. 动态性

    从各种角度看， Java 与 C 或 C++ 相比更加具有动态性。它能够适应不断发展 的环境库中可以自由地添加新方法和实例变量， 而对客户端却没有任何影响。在 Java 中找出运行时类型信息十分简单

    • 参考：
      • https://bbs.csdn.net/topics/370045736
      • https://www.iteye.com/blog/eric-weitm-1474640
    • 允许程序动态地装入运行过程中所需的类
