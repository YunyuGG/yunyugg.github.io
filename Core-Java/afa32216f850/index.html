第3章 Java的基本程序设计结构

3.1 一个简单的Java应用程序

  1. Java区分大小写，main不能写成Main
  2. 关键字public称为访问修饰符，用于控制程序的其他部分对这段代码的访问级别。第5章详细介绍。
  3. 关键字class表明Java程序中的全部内容都包含在这个类中，在这里只需要将类作为一个加载程序逻辑的容器，其中程序逻辑定义了应用程序的行为，Java应用程序中的全部内容都必须放置在类中。第4章详细介绍类。
  4. class关键字后面为类名。

当使用java ClasName运行已编译的程序时，Java虚拟机将从指定类中的main方法开始执行。

  1. 代码中的大括号**{}**划分程序的各个部分（称为块）
  2. 暂且不对static void分析，需要用记住每个Java应用程序都必须有一个main方法
  3. 在这里使用了System.out.对象并调用了它的println方法。

3.2 注释

  • 单行注释：//
  • 多行注释：/*和*/。【该类型注释不能嵌套】
  • 文档注释：/**开始，*/结束。会自动生成文档，参见第4章。

3.3 数据类型

​ Java是一种强类型语言，因此，必须为每一个变量声明一种类型。

​ 在Java中一共有8种基本类型：4种整型、2种浮点类型、1种用于表示Unicode编码的字符单元的字符类型char、1种用于表示真值的boolean类型。

Java有一个能表示任意精度的算术包，通常称为“大数值”（big number），但它并不是Java类型，而是一个Java对象。

3.3.1 整型

  1. int类型最常用，占4个字节。
  2. 当int长度不够时则选择long类型，占8字节。
  3. byte（1字节）和short（2字节）类型主要用于特定的应用场合，例如，底层的文件处理或者需要控制占用存储空间量的大数组。

在Java中，整型的范围与运行Java代码的机器无关，可移植性的体现。与此相反，C和C++程序需要针对不同的处理器选择最为高效的整型，这可能导致32位处理器上运行的程序在16位系统上运行发生整数溢出。Java的各种数据类型的取值范围是固定的，使得所有机器上都能得到相同的运行结果。

  • 长整型数值有一个后缀L或者l。十六进制数值有一个前缀0x或者0X。八进制有一个前缀0，例如010对应八进制中的8。显然八进制容易混淆，不推荐使用。
  • 从Java 7开始，加上前缀0b或0B就可以写二进制数，例如，0b1001就是9。Java 7开始可以为数字字面量加下划线，例如1_000_000表示一百万，下划线只是为了让人更易读，Java编译器会去除这些下划线。
  • Java没有任何无符号（unsigned）形式的int、long、short、byte类型。

3.3.2 浮点类型

浮点类型用于表示有小数部分的数值。

  1. double表示这种类型的数值精度是float类型的两倍。
  2. 大多数情况采用double类型，因为float类型的精度难以满足要求
  3. float类型的数值带有后缀F或f。没有后缀的浮点数值默认为double类型，double类型也有其后缀D或d。

可以使用十六进制表示浮点数值。例如：0.125=2-3可以表示成0x1.0p-3。

十六进制表示法中，使用p表示指数，而不是e。

注意：尾数采用十六进制，指数采用十进制（-3），尾数即小数点后面的数。

指数的基数是2而不是10。所以 0x1.0p-3 = 1.0乘以2的-3次方 = 1.0 *（1/8）=0.125

  • 所有的浮点数值计算都遵循 IEEE 754 规范。
  • 用于表示溢出和出错情况的三个特殊的浮点数值：
    • 正无穷大：Double.POSITIVE_INFINITY
    • 负无穷大：Double.NEGATIVE_INFINITY
    • NaN（不是一个数字的数字）：Double.NaN
    • 正整数除以0的结果为无穷大、0/0或者负数的平方根结果为NaN。

三个特殊的浮点数值常量（Float类也有）实际中很少遇到。

特别说明：判断一个数是否为NaN

if( x == Double.NaN)//is never true

所有的“不是一个数字”的值都认为是不同的，因此需要使用方法Double.isNaN(x)判断

  • 浮点数值不适用于无法接受误差的金融计算中，比如2.0-1.1结果输出不是0.9，double和float类型会存在数值误差，这种误差原因是浮点数值采用二进制系统表示，而二进制无法精确地表示分数1/10。因此，应该使用BigDecimal类。本章后续介绍。

3.3.3 char类型

char类型原本用于表示单个字符，但现在已经有所变化，见下一节。

  • char类型的字面量值需要用单引号括起来。例如：’A’是编码值为65所对应的字符常量。它与”A”不同，”A”是一个包含字符A的字符串。char类型的值可以表示为十六进制值，其范围从\u0000到\Uffff。例如：\u2122表示注册符号(™)。
  • 除了转义序列\u外还有一些用于表示特殊字符的转义序列。所有的转义序列都可以出现在字符字面量或者字符串中，例如’\u2122’或”Hello\n”。转义字符\u还可以出现在字符或者字符串之外，例如：main(String\u005B\u005D args)等同于String[]

Unicode转义序列会在解析代码之前得到处理。例如，”\u0022+\u0022”得到的字符串不是”+”，因为\u0022会在解析之前转换位”，因此实际上会是””+””，最终是一个空串。

隐蔽的错误：

  • 在注释中：// c:\users
  • 产生语法错误，因为\u后面未跟着4个十六进制数
    • 理解：因为它会在解析之前转义，即还没被看成注释之前

3.3.4 Unicode和char类型

想要弄清char类型，就必须了解Unicode编码机制。Unicode打破了传统字符编码机制的限制。

了解：

​ Unicode之前，已经有多种字符编码标准：美国的ASCII、西欧语言ISO 8859-1、中国的GB 18030和BIG-5等。多种标准导致的问题：①对于同一个代码值，在不同的编码方案下有可能对应不同的字母。②采用大字符集的语言其编码长度可能不同。例如一些字符用单字节编码，而另一些字符则需要两个或更多个字节。

​ Unicode设计目的就是解决上述问题。最初，人们认为两个字节的代码宽度足够对全世界所有字符进行编码。1991年发布了Unicode 1.0，一共65536个代码值（2个字节16位，即16位2进制数的个数为65536），然而实际上使用了不到一半的部分。

Unicode编码：使用数字从0开始表示字符【与计算机无关】，然后在计算机中认为采取2个字节足够表示，所以设计决定Unicode通过0到65536之间的数字来表示全世界所有字符【Unicode 1.0】，现在已经到4个字节了。

​ 设计Java的字符类型时，决定采用16位（char类型大小2个字节）的Unicode字符集，但遗憾的是，由于加入了汉语、日语、汉语导致Unicode字符超过了65536个（超过2个字节）。现在，Java中16位的char类型已经无法满足描述所有Unicode字符。

小结：

Java中char类型设计之初采用Unicode字符集，即char类型占用2个字节。

Unicode为世界上所有字符都分配了一个唯一的数字编号，这个编号范围在计算机中设定为从 0x000000 到 0x10FFFF(十六进制)。由最初 0x0000扩充到了0xFFFF。

​ 利用一些专业术语解释，从JavaSE5.0开始解决上述问题的基本方法：①码点：代码值，和编码表中一个字符对应。Unicode中码点采用十六进制书写，并加上前缀U+。②代码级别：Unicode的码点可以分成17个代码级别（U+16进制数分成17个级别）。码点U+0000到U+FFFF为第一个代码级别称为“基本的多语言级别”，包括经典的Unicode代码。其余的16个级别码点从U+10000到U+10FFFF，其中包括一些辅助字符。

为什么Unicode码点是17个代码级别：因为十六进制中0到10的差值为17，即U+0为第一级别，到U+1xxxx到U+10xxxx为剩余16级别。

Unicode超过65536后，仍然想通过两个字节的空间来存全部Unicode字符，所以可以采用UTF-16实现。

​ UTF-16编码采用不同长度的编码表示所有Unicode码点。①代码单元：在第一级别“基本的多语言级别”中，每个字符用16位表示，通常被称为代码单元。②替代区域：辅助字符采用一对连续的代码单元进行编码，这样构成的编码值落入“基本的多语言级别”中空闲的2048字节内，通常被称为替代区域[码点U+D800U+DBFF用于第一个代码单元，码点U+DC00U+DFFF用于第二代码单元]。这样我们通过码点可以识别出其在代码单元的位置和表达的意义。

https://blog.csdn.net/qq1337715208/article/details/104412642

UTF-16编码是Unicode的在计算机中存储的一种实现方式。

Unicode分级别后，会留下U+D800 ~ U+DFFF这块区域，大小2048，分为两部分，这两部分结合能够表达210*210=220大小的字符空间，足以满足全部Unicode字符。

​ - 为什么有剩余？理解：如同1和2组成12和21，因为Unicode预感到以后还要不断的扩充字符集，不希望每次都直接采用新的位数扩充，否则存储的时候会超过2个字节空间

Java中char类型为2个字节，采用UTF-16存储，所以U+D800 ~ U+DFFF这块区域内表示的字符无法使用char类型表示。【因为需要采用两个十六位二进制表示即占用4个字节】

建议：不要在Java程序中使用char类型，因为两个char来表示1个Unicode，导致String中char的数量不等于unicode字符的数量。str.char()可能出错？

3.3.5 boolean类型

​ Java中整型值和布尔值之间不能相互转换。

​ C++中0表示false，0以外的整型值布尔值为true。

3.4 变量

3.4.1 变量初始化

Java中声明一个变量后，必须先初始化再使用。

虽然有初始默认值，但是未初始化就是用会报错。

3.4.2 常量

关键字 final 表示这个变量只能被赋值一次。一旦被赋值之后，就不能够再更改了。习惯上, 常量名使用全大写。

类常量：在一个类中，定义于main方法外部，使用关键字static final定义类常量，可以供给类中多个方法使用，如果设置访问权限为public则可以供给其他类使用。

3.5 运算符

参与操作的两个数都是整数时，表示整数除法；否则表示浮点除法。

注意：整数被0除会产生异常，而浮点数被0除会得到无穷大或NaN。

3.5.1 数学函数与常量

  • 函数：
    • 平方根：Math.sqrt(x)
    • 幂运算：Math.pow(x,a)，x的a次幂
    • 求模运算：floorMod(num,mod)，但是如果是负数则会得到-1
  • 常量：Math.PI、Math.E

3.5.2 数值类型之间的转换

  • 合法转换：存储范围小的向存储范围大的转换，可以直接赋值形式转换。

  • 实箭头表示转换无信息丢失

  • 虚箭头表示可能有精度丢失的转换

  • 进行二元操作时，优先转换为其中优先级高的类型，再进行计算。

    • double>float>long>int

3.5.3 强制类型转换

  • 强制类型转换：存储范围大的向存储范围小的转换，转换时加括号给出转换的目标。
  • 浮点数转整数默认截取整数部分。
  • 如果超出目标类型的表示范围则会得到完全不一样的数。
