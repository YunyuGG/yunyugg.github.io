第4章 万能的搜索

第1节 深度优先搜索–不撞南墙不回头

1.全排序解决方法

  1. 问题形象化：假如我有1、2、3号牌，对应地面有1、2、3号盒子，每个盒子的只能放一张牌，有多少种方法？
  2. 约定：我决定每个盒子存放的顺序都是从1到3轮流存放
  3. 存放：走到1号盒子面前手里有123则放1，走到2号盒子面前，手里有23则放2，走到3号盒子面前手里只有3则放入3，此时手里的牌已经放完；收回3号盒子中的3号牌，要放入与之前不同的牌，手里只有3，只能往回走；收回2号牌，按照约定，这一次在2号盒子放3，3号盒子只能放2，放牌结束，然后再收回，直到收回1号盒子的1；按照约定这一次1号盒子放2，2号盒子按约定先放1有则放入，3号盒子先放1，没有，放2没有，放3有则放入，手里没牌，再次回收。依次类推.

代码思想：

发现自己之前做的便是这种思想，但不够具有逻辑性

  1. 如何往一个盒子里放牌？按顺序放，逐一放入：一个for循环

  2. a[step]表示当前走到第step个盒子前。问题：一张牌只能使用一次，因此使用数组book标记该张牌是否还在手中，如果没有该张牌则不能放入。

  3. 以上的代码表示了第step个盒子是如何放入一张牌的，我们要继续放下一个盒子，即step+1。如何处理第step+1个盒子呢？显然每一个盒子的处理方法是相同的。–因此，我们可以将放号牌的方法进行封装成一个dfs函数。

  4. 处理完第step个盒子后，要处理step+1个盒子

    • book[i]=0：表示回收扑克牌，因为上一步dfs(step+1);递归走完了才会执行到book[i]=0;
      • 一次尝试，需要往回走，因此设置book[i]=0收回扑克牌

  5. 最后一个问题：当最后一个盒子放完后，需要输出，然后停止。

2.全排序解决总结

  • 核心代码只有上面一段，但是却饱含深度优先搜索（Depth First Search,DFS）的基本模型
  • 理解深度优先搜索的关键在于解决“当下如何做”
  • 至于“下一步如何做”则与“当下如何做”是一样

第2节 解救小哈

1.问题描述

描述：找出最短路径，有障碍物的格子不能走

  1. 二维数组存储迷宫。小哼：(1,1)；小哈：(p,q)

  2. 约定尝试顺序：右下左上（顺时针）

  3. dfs函数：解决当前应该怎么办。小哼在某个点的处理：

    1. 检查是否找到了小哈，找到则输出
    2. 没有找到则找出下一步可以走的地方

2.问题分析

  • 参数：小哈的坐标p,q固定，小哼的坐标x,y需要传入，以及走的步数step

  1. 检查是否找到

  2. 没有找到小哈，进行下一步，右下左上四个方向对应x,y四个变化，用一个next数组表示下一步的路程

  3. 对于下一个坐标，需要判断是否到边界或者遇到了障碍物，以及判断是否已经走过，用一个book数组记录走过的坐标

  4. 如果坐标符合要求则执行下一步，step+1，且当尝试结束时需要回收坐标

第5节 宝岛探险

1.问题：求出降落点的岛的面积

  • 地图10x10：

2.解决思路

  • 广度优先：出队一个元素的同时入队其邻接元素

  • 深度优先：递归

    • 进行了着色：

    • 深度优先方法一：直接保存当前点再寻找判断下一个合适的点

    • 深度优先方法二：先判断当前点是否符合，然后再进入一个点

  • 扩展：获取一共多少个小岛

    • 从(1,1)开始到(n,m)，遍历大于0的点【通过标记避免重复判断】

    • 需要改用深度优先方法二，才可以进行着色判断

3.总结

  • 该问题实际上就是求一个图的子图
  • 求小岛个数等同于求子图个数，该算法即FloodFill漫水填充法（种子填充法）
  • 应用：图像识别、物体识别。
    • 如：画图软件的油漆桶工具、Photoshop的魔术棒选择工具
    • 具体算法：查找种子点周边的点将与种子颜色相近的点（设定阙值决定相近与否）入队作为新种子，并对新入队的种子做同样的扩展
